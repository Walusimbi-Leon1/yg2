## Pop Party Discord Activity Fix — Round 2 (OAuth)
- First fix (heartbeat + polling) didn't work — game still didn't show live players in Discord
- **Real root cause found:** OAuth consent screen was being suppressed
  - `prompt: "none"` in authorize() silently skipped the authorization modal
  - Users were falling through to anonymous "Player" IDs with no real Discord identity
  - Without real user data, the app had no stable player IDs for Firebase
- **discord.js rewritten** with proper OAuth2 code flow:
  1. Try silent `authenticate()` first (works if already authorized)
  2. If that fails, call `authorize()` — **no prompt param** so the OAuth consent modal shows
  3. Exchange authorization code for access token via `/api/exchange`
  4. Call `authenticate({ access_token })` to get real Discord user info
  5. Use real Discord username + snowflake ID
- **Cloudflare Pages Function** created at `functions/api/exchange.js`:
  - Proxies the code → token exchange with Discord's `/api/v10/oauth2/token`
  - Keeps `client_secret` server-side (never exposed to users)
  - Environment variables: DISCORD_CLIENT_ID, DISCORD_CLIENT_SECRET, DISCORD_REDIRECT_URI
- Created `.gitignore` for the Pop Party repo
- Pushed everything to Pop Party repo
- **User needs to configure:**
  1. Discord Developer Portal → OAuth2 → add redirect URL `https://pop-party.pages.dev/`
  2. Cloudflare Pages → add DISCORD_CLIENT_ID, DISCORD_CLIENT_SECRET, DISCORD_REDIRECT_URI
- User reported game still not working after first fix; pending test of OAuth fix

### Hindsight Browser Testing Results
- Tested in remote browser via CDP debugging — confirmed game hangs on splash screen "Connecting..." when loaded outside Discord's iframe
- **Root cause**: `new DiscordSDK(clientId)` throws synchronous error `frame_id query param is not defined` because the page isn't inside Discord's sandbox
- **try/catch handles the error** but game doesn't progress past splash — the lobby UI never renders because Discord auth is prerequisite
- This is **expected behavior** — Hindsight is a Discord Activity and requires the Discord iframe context to function fully
- The game will work properly when launched from Discord's Activity Launcher

### Hindsight Discord SDK Timeout Fix
- `discordSdk.ready()` hangs forever when called outside Discord's iframe (never resolves/rejects)
- Added 5-second `withTimeout` wrapper around `ready()` so it falls to browser mode after timeout
- Without this fix, the page would hang indefinitely on "Connecting..."
