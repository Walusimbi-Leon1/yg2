## Pop Party Discord Activity Fix — Round 2 (OAuth)
- First fix (heartbeat + polling) didn't work — game still didn't show live players in Discord
- **Real root cause found:** OAuth consent screen was being suppressed
  - `prompt: "none"` in authorize() silently skipped the authorization modal
  - Users were falling through to anonymous "Player" IDs with no real Discord identity
  - Without real user data, the app had no stable player IDs for Firebase
- **discord.js rewritten** with proper OAuth2 code flow:
  1. Try silent `authenticate()` first (works if already authorized)
  2. If that fails, call `authorize()` — **no prompt param** so the OAuth consent modal shows
  3. Exchange authorization code for access token via `/api/exchange`
  4. Call `authenticate({ access_token })` to get real Discord user info
  5. Use real Discord username + snowflake ID
- **Cloudflare Pages Function** created at `functions/api/exchange.js`:
  - Proxies the code → token exchange with Discord's `/api/v10/oauth2/token`
  - Keeps `client_secret` server-side (never exposed to users)
  - Environment variables: DISCORD_CLIENT_ID, DISCORD_CLIENT_SECRET, DISCORD_REDIRECT_URI
- Created `.gitignore` for the Pop Party repo
- Pushed everything to Pop Party repo
